# 三分鐘短講提案稿

我們小組的題目是：校園宿舍報修、派工與維修追蹤系統。

目標使用者是：宿舍學生、宿舍管理員與維修人員。

目前的問題是：宿舍學生目前多靠電話、訊息或紙本報修，導致維修狀態不清楚、派工效率低，維修人員也缺少統一的任務與回報機制。

本系統希望改善：整合報修申請、維修派工與維修狀態追蹤，讓報修流程更透明、任務分派更清楚、完成回報更可追溯。

本期範圍內包含：報修單建立與編輯、報修狀態查詢與進度追蹤、維修派工與派工人員狀態更新、維修完成回報與關閉流程、以及基本角色與權限管理。

本期範圍外包含：實體維修零件採購流程、真實金流付款、正式校內單一登入整合、正式部署到學校伺服器、手機原生 App 開發。

期末原型會展示三個任務：

1. 宿舍學生提交報修單並查看報修狀態。
2. 宿舍管理員在待處理報修清單中分派維修人員並更新任務狀態。
3. 維修人員完成維修並回報結果，系統更新完成狀態。

目前最大的風險是：原型範圍若定義太大，會導致 5 週內難以完成。

我們的應對方式是：將採購、金流、正式登入與正式部署移到範圍外，專注於報修、派工與狀態追蹤的核心流程。
